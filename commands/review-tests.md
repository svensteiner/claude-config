# Test Coverage Review

Analysiere die Test-Abdeckung der aktuellen Änderungen.

## Anweisungen

1. Prüfe mit `git diff` welche Dateien geändert wurden
2. Nutze den `pr-review-toolkit:pr-test-analyzer` Agent um die Test-Coverage zu analysieren:
   - Fehlende Tests identifizieren
   - Edge Cases prüfen
   - Test-Qualität bewerten

3. Gib konkrete Vorschläge welche Tests hinzugefügt werden sollten
