Orchestrate multiple agents for complex tasks.

Usage: /multi-orchestrate <task description>

## Available Agents
| Agent | Model | Purpose |
|-------|-------|---------|
| orchestrator | opus | Coordinate workflows |
| code-architect | opus | Architecture design |
| code-explorer | sonnet | Codebase analysis |
| task-executor | haiku | Fast implementation |
| quality-guardian | sonnet | Code quality |
| security-sentinel | opus | Security analysis |

## Process
1. Analyze task requirements
2. Select appropriate agents
3. Create execution plan
4. Run agents (parallel when possible)
5. Aggregate results

## Agent Selection
- Design/Architecture → code-architect
- Understanding code → code-explorer
- Implementation → task-executor
- Quality check → quality-guardian
- Security → security-sentinel
