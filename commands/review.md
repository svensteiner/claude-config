# Quick Code Review

Führe einen schnellen Code-Review der aktuellen Änderungen durch.

## Anweisungen

1. Prüfe mit `git diff` welche Dateien geändert wurden
2. Nutze den `/code-review:code-review` Slash Command für den vollständigen Review mit 5 parallelen Agenten:
   - CLAUDE.md Compliance
   - Bug Detection
   - Historical Context
   - PR History Analysis
   - Code Comments Quality

3. Fasse die Ergebnisse kurz zusammen mit:
   - Kritische Issues (MUSS gefixt werden)
   - Warnungen (SOLLTE gefixt werden)
   - Hinweise (KANN verbessert werden)
